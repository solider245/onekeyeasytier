# OneKeyEasyTier Web管理器 - 技术方案设计

## 项目概述

基于Go语言开发的EasyTier Web管理器，提供图形化界面管理EasyTier网络配置和服务，解决现有脚本的系统检测不足和进程冲突问题。

## 架构设计

### 系统架构
```
┌─────────────────────────────────────────────────────────────┐
│                    Go Web管理器                            │
├─────────────────────────────────────────────────────────────┤
│  HTTP Router   │  配置管理器   │  服务控制器  │  系统检测器  │
├─────────────────────────────────────────────────────────────┤
│  静态文件服务  │  RESTful API  │  WebSocket  │  系统命令执行 │
└─────────────────────────────────────────────────────────────┘
                            │
                ┌─────────────────────────┐
                │      系统层面           │
                ├─────────────────────────┤
                │  配置文件  │  二进制文件  │
                │  服务进程  │  系统服务    │
                └─────────────────────────┘
```

### 核心模块

#### 1. 系统检测器 (SystemDetector)
- 操作系统检测 (Linux/macOS/Windows/OpenWrt)
- 架构检测 (x86_64/aarch64)
- 服务管理器检测 (systemd/launchd/procd)
- 现有组件检测 (二进制文件/配置文件/服务进程)

#### 2. 配置管理器 (ConfigManager)
- TOML配置文件读写
- 配置验证和格式化
- 配置备份和恢复
- 配置模板管理

#### 3. 服务控制器 (ServiceController)
- 服务启停控制
- 配置重载
- 进程监控
- 日志管理

#### 4. Web界面 (WebInterface)
- 配置编辑器
- 状态监控面板
- 服务控制界面
- 实时日志查看

## API设计

### RESTful API端点
```
GET    /api/status              # 获取系统状态
GET    /api/config              # 获取当前配置
POST   /api/config              # 保存配置
POST   /api/service/start       # 启动服务
POST   /api/service/stop        # 停止服务
POST   /api/service/restart     # 重启服务
POST   /api/service/reload      # 重载配置
GET    /api/logs                # 获取日志
GET    /api/templates           # 获取配置模板
```

### WebSocket端点
```
WS /ws/status      # 实时状态推送
WS /ws/logs        # 实时日志推送
```

## 数据结构

### 系统信息
```go
type SystemInfo struct {
    OS           string            `json:"os"`
    Arch         string            `json:"arch"`
    ServiceMgr   string            `json:"service_manager"`
    Components   ComponentStatus   `json:"components"`
    Services     []ServiceInfo     `json:"services"`
}

type ComponentStatus struct {
    ConfigFile   string   `json:"config_file"`
    BinaryPath   string   `json:"binary_path"`
    CliPath      string   `json:"cli_path"`
    ConfigExists bool     `json:"config_exists"`
    BinaryExists bool    `json:"binary_exists"`
    CliExists    bool     `json:"cli_exists"`
}
```

### 配置结构
```go
type EasyTierConfig struct {
    Network struct {
        Name     string `toml:"name"`
        IP4Cidr  string `toml:"ipv4_cidr"`
        IP6Cidr  string `toml:"ipv6_cidr,omitempty"`
    } `toml:"network"`
    
    Peers []struct {
        URI string `toml:"uri"`
    } `toml:"peers"`
    
    DHCP struct {
        Enabled bool   `toml:"enabled"`
        IP4Pool string `toml:"ipv4_pool,omitempty"`
        IP6Pool string `toml:"ipv6_pool,omitempty"`
    } `toml:"dhcp"`
}
```

## 部署方案

### 单机部署
```bash
# 编译
go build -o easytier-web cmd/web/main.go

# 运行
sudo ./easytier-web
```

### Docker部署
```bash
# 构建镜像
docker build -t easytier-web .

# 运行容器
docker run -d --name easytier-web \
  -p 8080:8080 \
  --privileged \
  -v /etc/easytier:/etc/easytier \
  easytier-web
```

### 系统服务部署
```bash
# 安装为系统服务
sudo ./scripts/install.sh

# 启动服务
sudo systemctl start easytier-web
```
